# Rime XSB (声笔自然)

极简、稳定、高效的声笔输入模式。

这套配置专注于 `声笔自然 sbzr` 方案，通过优化词库结构和引入强大的辅助工具，提供极致的中文输入和维护体验。

## 核心特性

- **稳定优先**：移除所有运行时 Lua 依赖（`librime-lua`），避免因脚本加载异常导致输入法崩溃。
- **高效维护**：配套专用的 Chrome 词库编辑扩展 (Notepad Extension)，支持多标签编辑和高级语法高亮。
- **结构简洁**：词库统一归档于 `dicts/`，主入口文件仅负责逻辑链接。

## Notepad 词库编辑扩展 (SBZR Extension)

集成在 Chrome 插件中的词库管理工具，具备以下高级特性：

- **VIM 仿真模式**：支持 Normal/Insert 模式切换，提供 `h/j/k/l`、单词跳转、`dd`、`u`、`Ctrl+r` 等原生 VIM 操作感。
- **高性能语法高亮**：
  - **虚拟渲染技术**：仅处理当前视口可见内容，支持数十万行超大词库无卡顿编辑。
  - **全能高亮引擎**：智能识别 Rime 词库格式、YAML 头部以及 Python, JS, Java 等主流编程语言。
  - **空白字符显示**：可视化 Tab 和空格，方便检查词库对齐。
- **Gedit 深色系风格**：基于现代深色主题设计，视觉舒适，专注度高。
- **数据持久化**：自动保存工作区状态，刷新页面或切换标签后能完美还原内容和光标位置。
- **双端同步**：支持将扩展内的输入历史同步回 Rime 系统数据库，实现习惯互通。

## 输入方案与操作

### 方案文件
- `sbzr.schema.yaml`: 主方案逻辑
- `sbzr.custom.yaml`: 个人偏好定制

### 候选选择
- `Space`: 选择第 1 候选
- `1-5`: 选择第 2-6 候选
- `Tab`: 向后移动候选
- `Shift+Tab`: 向前移动候选

### 快捷编辑
- `Shift+BackSpace`: 清空当前编码
- `Shift`: 切换输入法启用/禁用

## 词库管理规则

1. **4 码单字 2 码匹配**：通过单字前缀词库，实现输入 2 码即可匹配高频单字，不依赖补全逻辑。
2. **频率与长度对齐**：相同编码下，优先按权重和字数（短词优先）排序，确保输入效率。
3. **词库归口**：
   - `dicts/sbzr.dict.yaml`: 主词库
   - `sbzr.chrome.extension/dicts/fixed.dict.yaml`: 固定位词库
   - `sbzr.chrome.extension/dicts/zdy.dict.yaml`: 自定义长句/高频词

## 快速重建

如修改了配置文件或词库，请运行：
```bash
./rebuild.sh
```
该脚本会自动清理 `build/` 缓存，重新编译并链接所有配置。

让Win10/11中文输入法使用日式键盘布局（适用于日版Macbook）
背景
从日本带回来一台MacBook，键盘是日式布局。MacOS下无论哪种输入法都适配得很好，但用Boot Camp（启动转换）安装了Windows10后，Windows下的中文拼音输入法仅支持美式键盘，导致输入的符号和键盘上的标识完全是错位的，造成了很大不便。

解决方法
如果不想折腾就直接用虚拟键盘或者外接。

Windows内置了日式键盘布局，但除日文输入法外，对其他输入法并没有提供设置键盘布局的选项。所以需要通过修改注册表，使系统键盘布局变为默认日文布局。

打开注册表，前往[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Keyboard Layouts\00000804]，找到键Layout File，此时键值应为默认值KBDUS.DLL。如果你用的是Win11或Win10（1809）以后的版本，则将该值修改为KBD106.dll，否则修改为KBDJPN.DLL。

修改完成后重启系统，中文输入法的键盘布局就变为日式布局了，和键盘标识可以一一对应了。

关于如何输入DEL
日式键盘的Macbook上有一个“delete”按键，然而按下它后实际输入的内容是退格。要想输入DEL，需要按“fn + delete”。
